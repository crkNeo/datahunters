## 雷達型(星軌 / 超新星)

兩本共用同一組雷達訊號與 TP/SL 公式,差別只在濾網。

**TP/SL 公式**(`radar.go` `entryLevels`,R = 最近 **12 根**的擺盪區間 = 該窗最高−最低):

```
做多:  TP = 現價 + 0.618 × R      SL = 現價 − 0.5 × R
做空:  TP = 現價 − 0.618 × R      SL = 現價 + 0.5 × R
```
> `tpMult, slMult = 0.618, 0.5` — 註解註明是回測最佳(勝率與期望值均勝過原本的 1.382 遠端延伸)。
> ⚠️ 因此**雷達單的先天盈虧比僅約 1.24R**,這是「分批止盈必須用比例、不能用 R 倍數」的原因。

**共用條件**:最多同時 15 單(`paperMaxOpen`)、同幣同向去重、**逾時 24h** 市價結算、`Pos` = 現價在 12h 區間的位置(0=底,1=頂)。

| | `main` 星軌 | `gamble` 超新星 |
|---|---|---|
| 分數門檻 | **≥55** | **≥50** |
| 需新鮮金叉 | ✅(要先在門檻下方「上膛」) | ❌ 直接追 |
| 冷卻 | **4h** | **1h** |
| 止損距離濾網 | — | **≤12%** |
| 分批止盈 | tpMomentum | tpMomentum |

- **分數門檻 50 是回測結論**:45–49 桶淨虧、50–59 最佳。不是越高越好。
- 程式另有 `skipNY` / `requireAlign` / `requireFuel` / `requireEMA` / `posGate` 等濾網欄位,**目前兩本都沒啟用**。`posGate`(不追竭盡:多單 12h 區間位置 >0.9 / 空單 <0.1 不進)是已撤掉的 gambleB 留下的現成實驗素材。

## `emaonly` 銀河 · 1h · 多空

**⚠️ 舊文件說「EMA5/20 交叉 + 15m EMA200 同側」是錯的** —— 全套件沒有任何 `15m`,也沒用 EMA200。

```
進場(1h 收盤,狀態由 false→true 的那一刻觸發):
  做多:EMA5 > EMA20  且  收盤 > EMA50
  做空:EMA5 < EMA20  且  收盤 < EMA50

止損:進場前 20 根的擺盪極值(多單=最低低點 / 空單=最高高點)
止盈:1:1 R(R = |進場 − 止損|)
逾時:無 —— 開著直到 TP 或 SL
```
- 每個「訊號小時」同幣只進一次(`sigHour` 去重),無時間冷卻。
- 分批止盈:tpMomentum。
- 管理員手動出場記為 **`逾時`**(其他策略是 `動能衰弱`)。

## `conv` 冥王星 · 4H · 多空

```
進場(每根 4H 收盤評估):
  1. 價格與 EMA200 同側
  2. 連續 4 根橫盤:該 4 根區間 ≤ 3.0 × ATR(14)
  3. 該 4 根「靠 EMA200 那側的極值」距 EMA200 ≤ 1.5 × ATR(動態空間,取代固定 3%)
  4. 濾網:盈虧比(TP距/SL距)≥ 1.5 才開倉

止損:4 根盤整區極值 ∓ 0.3 × ATR(結構止損 + 掃針緩衝)
止盈:VRVP 近似 —— 沿高低價把成交量攤進 60 個價格分箱(回看 96 根),
      取進場價上方(多)/下方(空)最近的「高量節點」
      (該分箱量 ≥ 0.6 × POC 量)
逾時:60 根(≈10 天)市價結算
```
- 分批止盈:tpMomentum。宇宙 = 銀河幣池(`emaCoins`)。

## 均值回歸組(admin)

都是:每根收盤評估、**同幣冷卻 4 根**、宇宙 = 銀河幣池。
**止損距離濾網 ≤10%**(`maxSLPct`)只套在 rsifade / bollfade / meanrev —— **布乖v2 沒有**(見下)。

### `rsifade` 逆勢超買空 · 30m · **僅做空**
```
進場:RSI(3) > 90  且  收盤 < EMA200(空頭中的反彈)
止損:進場 + 2.5 × ATR(14)
止盈:進場 − 2.0 × ATR(14)
逾時:16 根
分批:tpMeanRev(50/30/20)
```

### `bollfade` 布林重回 · 1h · 多空
```
進場:前一根收盤在 布林(20, 2σ) 通道【外】,本根收盤回到通道【內】,且與 EMA200 同側
  做空:前根 收盤>上軌、本根 收盤≤上軌、收盤<EMA200
  做多:前根 收盤<下軌、本根 收盤≥下軌、收盤>EMA200
止損:進場 ∓ 2.5 × ATR(14)
止盈:布林中軌(SMA20)
額外濾網:盈虧比必須落在 [0.4, 3.0]
逾時:24 根
分批:tpMeanRevFront(60/25/15,位置 0.45/0.60)
```

### `meanrev` 乖離回歸 · 1h · 多空
```
進場:收盤偏離 EMA20 超過 2.0 × ATR(14),且與 EMA200 同側
  做多:收盤>EMA200 且 (收盤−EMA20) < −2 ATR   (上升趨勢中的回檔)
  做空:收盤<EMA200 且 (收盤−EMA20) > +2 ATR   (下降趨勢中的反彈)
止損:進場 ∓ 3.0 × ATR(14)
止盈:EMA20
逾時:24 根
分批:tpMeanRevFront(60/25/15,位置 0.45/0.60)
```


### `bgv2` 布乖v2 · 1h + 4h · **僅做空** · 雙腿家族

**一個分頁、一個開關、兩條腿**(DB book 各自為 `bgv2dev` / `bgv2boll`)。
⚠️ **照回測規格原樣上線:單段止盈(不分批)、無 maxSLPct 濾網**(SL 本就刻意放寬到 4 ATR,再加 10% 濾網會砍掉回測納入的單)。

**通用設定(兩腿共用)**
```
方向:只做空
市場過濾:收盤 < EMA200(各腿用自己週期的 EMA200)
止損:進場 + 4.0 × ATR(14)   ← 寬止損,扛插針
時間出場:持滿 64 根 → 收盤平倉
盈虧比閘門:RR 落在 [0.4, 3.0]
冷卻:4 根
同幣互斥:同一幣種同時只允許家族一個倉位(誰先觸發誰佔位,另一腿跳過)
同根同觸止損與目標 → 一律算止損(microRun 先判 SL,保守)
倉位建議:每筆風險 ≤ 0.5% 資金(歷史 maxDD 約 20–35R)
```

**腿 1 — 乖離回歸(1h)**
```
進場:① 收盤 < EMA200  ② (收盤 − EMA50) / ATR > +2.0  ③ RR 閘門
止盈:進場當下的 EMA50        逾時:64 根 ≈ 2.7 天
```
> 一句話:空頭趨勢裡出現過度反彈(高出 EMA50 兩個 ATR 以上),賭它衰竭、幾天內跌回 EMA50。

**腿 2 — 布林重回(4h)**
```
進場:① 收盤 < EMA200
      ② 前一根收盤 > 布林(50, 2σ) 上軌(衝出通道)
      ③ 本根收盤跌回上軌內、且仍在中軌(SMA50)上方(收回但未跌過頭)
      ④ RR 閘門
止盈:進場當下的中軌(SMA50)   逾時:64 根 ≈ 10.7 天
```
> 與既有 `bollfade` 的差異:4h 而非 1h、50 期布林而非 20 期、只做空、且多要求「仍在中軌上方」。

**⚠️ 家族鎖的實作**:兩腿共用一個 `famMu`,`microRun` 的鎖序固定為 **famMu → 自己的 mu → 手足的 mu**。
因為每條腿都先搶 famMu,同一時間只有一個 goroutine 會持有兩把書鎖,不可能成環;`microMarkTick`/`microState` 只取單一 mu,也無法閉環。已用 8 goroutine × 300 輪交錯上鎖實測無死鎖。

**腿1 的消融證據(為什麼是這些數字)**

| 決定 | 證據 |
|---|---|
| 均值用 **EMA50** 不是 EMA20 | 快均線只抓到幾小時的抖動,長樣本費後歸零;慢均線的反彈幅度夠大 |
| 止損 **4 ATR** 不是 3 | 2.0→4.0 單調改善,窄止損死於插針 |
| 持倉 **64 根** 不是 24 | 4→48 單調改善、48→96 趨平;v1 常在回歸完成前被砍 |
| 目標用 **EMA50 本身** 不用固定 R | 出場消融:EMA 目標 > 固定 2 ATR > 純時間出場 |
| **EMA200 濾網不能拿掉** | 拿掉後山寨空單 test 由 +0.043 → −0.023 |
| **只做空** | 多頭鏡像(跌破 EMA50 太深接多)全資料集無邊際 |
| **不等確認棒** | 等收盤回轉只換來勝率 +5pp,期望值沒變好 |

> ⚠️ **RR 閘門的下限對腿1 永遠不會觸發**:條件 2 已保證乖離 > 2 ATR,故 RR = 乖離/(4 ATR) > 0.5 > 0.4。腿1 只有上限生效(擋掉乖離 > 12 ATR 的極端值)。**腿2 則兩邊都有效**(下限 = 要求 收盤−中軌 > 1.6 ATR)。
> 📊 腿1 實測訊號率:16 幣 × 52 天 = 68 個訊號 ≈ **每天 1.3 個**。

## `bollema` 布林EMA · 4H · 多空 · 突破蓄勢

賭的是**突破後盤整、再啟動**,不是追突破。**單段止盈(1:3 RR)、不分批。**

**指標**(全 4H、收盤判斷):布林(20, 2σ) 中軌 = SMA20 ｜ EMA50(趨勢過濾,取代原版 1H EMA200,實測等價)｜ ATR(14)

**進場【做多】**(空單完全鏡像),四條依序全成立 → **K2 收盤市價進場**:
```
① 順大勢 :K2 收盤 > 4H EMA50
② 突破K  :某根收盤由下往上站上中軌(前一根收盤 ≤ 中軌、這根 > 中軌)
③ 蓄勢K1 :下一根收盤守在中軌上方,且 ≤ 突破K收盤 × 1.02
④ 蓄勢K2 :再下一根收盤守在中軌上方,且 ≤ 突破K收盤 × 1.02(累計漲幅 ≤ 2%)
```
> 這是本檔唯一的**三根 K 序列**訊號(其他都只看最後一根)。判斷落在最後一根收盤 = K2:
> `cs[n-4]`=突破K前一根、`cs[n-3]`=突破K、`cs[n-2]`=K1、`cs[n-1]`=K2。
> ⚠️ 原版的過濾 A(%B/帶寬喇叭口)與 過濾 B(長影線)**刻意不實作** — 消融證實是負貢獻。

**出場**(先到先出)
```
止損:中軌 − 1.5 × ATR(14)
止盈:進場價 + 3 ×(進場價 − 止損價) → 1:3 盈虧比
逾時:180 根(30 天)→ 收盤平倉(極少觸發)
冷卻:同幣 3 根
```
> 實作細節:先把 entry/SL 各自 `roundPx` 取整,**再由取整後的值導出 TP**——否則存下來的三個
> 數字彼此不是精準的 1:3(TP 由未取整的 SL 導出,顯示會對不上)。

**🛡 保本位 — 僅通知,不動止損**(`microBook.beAt = 0.3`)
```
價格首次觸及「進場 + 0.3 ×(止盈 − 進場)」→ 標記 BEHit、記下 BEPrice、發推播
⚠️ 止盈止損完全不變 —— 這只是提示該單已覆蓋風險,不是分批止盈的棘輪
```
- 閂鎖(latched):只觸發一次,不會因回檔而解除。
- **持久化**(`paper_trades.be_hit` / `be_price`):持倉可達 30 天,重啟一定會發生,不存會掉狀態。
  欄位沿用既有的 `information_schema` 遷移機制自動 `ALTER TABLE`。
- 判斷放在 `stepTP` **之前**:若同一 tick 直接衝到出場,仍要先記下曾走到過保本位。
- 前端在「進度」欄顯示藍色 `🛡 已達保本位 <價>` 徽章。

## 分批止盈引擎(`multitp.go`)

原本的 TP 變成**最終目標 TP3**;TP1/TP2 依「進場→TP3 距離」的比例擺放。
以 `Filled`(已平倉位比例)/ `Realized`(已鎖利%)記帳,`closeTrade` 混合結算,狀態進 DB。

```
TP1 = 進場 + a × (TP3 − 進場)     觸及 → 平 w1,止損移到【保本】= 進場 × (1 ± 0.0005)
TP2 = 進場 + b × (TP3 − 進場)     觸及 → 平 w2,止損移到【TP1】
TP3 = 原始止盈                    觸及 → 平掉剩餘 w3
若 |TP3−進場|/進場 < 0.8%(minSplitPct)→ 不分批,退回單段 TP
```

| 預設 | 用在 | 位置 a / b | 分批 w1/w2/w3 |
|---|---|---|---|
| `tpMomentum`(順勢組) | 星軌 / 超新星 / 銀河 / 冥王星 | 0.40 / 0.70 | 40 / 30 / 30 |
| `tpMeanRev`(回歸組) | 逆勢超買空 | 0.30 / 0.60 | 50 / 30 / 20 |
| `tpMeanRevFront`(回歸·前置) | 布林重回 / 乖離回歸 | **0.45** / 0.60 | **60 / 25 / 15** |
| (無) | **布乖v2**(兩腿) | — | 單段止盈,不分批 |

> `rMult` 模式(a/b 當 R 倍數)已實作但**未使用**:雷達 RR 僅 1.24R,裝不下 1.5R 的 TP2。
