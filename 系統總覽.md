# datahunter 系統總覽

自架的加密貨幣合約數據看板。後端 Go(純記憶體、無資料庫)+ 前端 Vue 3 / Vite。
分析對象:Binance USDT 永續合約。

---

## 一、整體架構

```
Binance / OKX / Deribit  ──(公開 API)──►  Go 後端(記憶體快取)  ──/api/*──►  Vue 前端(單頁、分頁切換)
```

- **後端**:`internal/cache`(快取 + 計算)、`internal/scorer`(評分器)、`internal/indicator`(技術指標)、`internal/exchange`(交易所 client)、`internal/api`(HTTP 路由)。
- **無資料庫**:全部存記憶體,**重啟資料即清空**(snapshot 尚未實作)。
- **背景定時更新**(`cmd/server/main.go`):

| 工作 | 間隔 | 內容 |
|------|------|------|
| `Refresh` | 60 秒 | 追蹤幣的方向評分(OI 儀表板) |
| `Radar` | 90 秒 | 全市場爆發雷達掃描 |
| `PaperTick` | 30 秒 | 模擬倉(紀律 + 賭博)開倉/監控/平倉 |
| `Options` | 3 分鐘 | BTC/ETH 選擇權儀表(Deribit) |

## 二、資料來源(全部免費公開端點)

| 來源 | 用途 |
|------|------|
| **Binance fapi**(合約) | K 線(含**主動買量** → 真實 CVD)、OI 歷史、資金費率歷史、多空帳戶比、大戶持倉、溢價、全市場 ticker、exchangeInfo(分辨幣/個股) |
| **Binance spot**(現貨) | 現貨 K 線(回測用) |
| **OKX** | candles、當下資金費率、**當下** OI(無歷史) |
| **Deribit** | DVOL 隱含波動率指數 + **選擇權鏈即時**(BTC/ETH:skew、期限結構、Max Pain、Put/Call、OI 牆) |

> ⚠️ **資料天花板**:Binance 的 OI 歷史 API 上限只有**最近 30 天**(實際抓 ~20-25 天),且是「滾動視窗」。這是為何只能在「單一行情」回測,也是 snapshot 的意義(往未來累積、突破 30 天牆)。

## 三、兩套獨立的分數系統(最重要的差異)

整個系統有**兩個完全不同的分數**,服務不同目的,**不要混淆**:

| | **OI 方向評分**(±20) | **爆發雷達 點火分**(≥55) |
|---|---|---|
| 問的問題 | 「**往哪個方向**走?」 | 「**會不會劇烈噴/崩**?」 |
| 性質 | 方向 + 強度(-100~+100) | 爆發強度(0~100,無關方向品質) |
| 範圍 | 追蹤的 ~36 個幣 | **全市場** ~300-400 個幣 |
| 用在 | OI 儀表板、數據訊號、推薦卡、訊號紀錄 | 爆發雷達、模擬倉、賭博單 |
| 核心因子 | OI 1h + CVD + 結構 + 動能 + 費率 + 多空比 | OI 堆積 + 量能爆發 + 鯨魚 + 加速 + CVD |
| 方向判定 | 各因子加權(含反指標修正) | OI 堆積方向(CVD 決勝負) |

## 四、OI 方向評分器(±20 系統核心)

`internal/scorer/detail.go`。同一套評分器同時供:OI 儀表板、推薦卡、個股明細、數據訊號——**三者永遠一致**。

### 因子與權重(`DefaultDetailWeights`)

| 因子 | 上限 / 半飽和 | 說明 |
|------|------|------|
| **市場結構**(OI 1h + CVD) | OI 15/1.5、CVD 15/8 | 頭號因子。OI 方向 × 主動買賣壓 |
| **價格結構**(HH/HL) | ±8 | 高點高/低點低的型態 |
| **24H 動能** | 10/2 | |
| **資金費率** | 10/0.05 | **反指標**(費率極正=多頭擁擠→偏空) |
| **多空比**(散戶) | 8/15 | **反指標**(散戶過度做多→偏空) |
| ~~1H 動能~~ | **0(停用)** | 回測證實是**反指標** |
| ~~相對強弱~~ | **0(停用)** | 回測證實是**反指標** |

- **流動性抑制**:`總分 = 原始分 × min(1, 24h量/100M)` — 薄量幣分數往 0 壓(回測:<100M 表現差很多)。
- **門檻**:`|總分| ≥ 20` 才標多/空(`BiasThreshold`);評等 = 總分 / 8(0~10 點)。
- **平滑飽和曲線** `smoothSat`:避免單一因子暴衝。

## 五、爆發雷達 點火分(≥55 系統)

`internal/cache/radar.go`。全市場(24h量 > $1M、扣穩定幣)併發掃描(10 workers)。

### 點火分組成(`earlyScore`)

```
量能爆發   clamp((volSpike-1)*6, 0, 40)   ← 最大權重
OI 堆積    clamp(|oiAccum|*1.8, 0, 30)    ← 最強訊號
鯨魚足跡   clamp((whale-1)*1.0, 0, 12)    ← 次要
價格加速   clamp(|accel|*1.0, 0, 12)
CVD       clamp(|cvd|*0.4, 0, 8)          ← 僅當與加速同向才計
× 早期性   clamp(1 - |24h漲跌|/68, 0.3, 1) ← 已噴一大段的降權
```

- **方向**:`OI 堆積 ≥ 0 → 看噴(pump)`,接近 0 時用 CVD 決勝負。**不是用動能判方向**(回測:OI 方向 57% 勝過動能 51%)。
- **進場/出場參考**:進場 = fib 0.382 回檔位、觸發 = 突破位;**TP = 現價 ± 0.618×R、SL = 現價 ∓ 0.5×R**(R = 近期擺盪幅度,回測網格最佳)。

## 六、各分頁說明

| 分頁 | 用的分數 | 資料 / 內容 |
|------|---------|------|
| **市場列表**(list) | — | 全市場 ticker,可按量/漲/跌排序 |
| **OI 儀表板**(oi) | ±20 方向評分 | 追蹤幣的評分/方向/品質/OKX%/OI 1h%/CVD%/費率 |
| **數據訊號**(signals) | ±20 方向評分 | 只列 `|評分|≥20` 的可進場訊號;**套用過濾層**(見下) |
| **訊號紀錄**(scorelog) | ±20 方向評分 | 記錄分數**穿越** ±20 的時間點(只記穿越、非啟動即記),供回溯 |
| **爆發雷達**(radar) | 點火分 ≥55 | 全市場潛在噴/崩,含現價、點火分、進場參考。幣/個股分開 |
| **訊號追蹤 / 模擬倉**(paper) | 點火分(紀律) | 自動模擬交易:紀律版 |
| **賭博單**(gamble) | 點火分(寬鬆) | 自動模擬交易:賭博版(A/B 對照) |
| **個股**(stocks) | 點火分 | 股票代幣(EQUITY)單獨呈現,不混入幣 |
| **BTC/ETH 選擇權**(majors) | 選擇權(Deribit) | 主力幣深度分析,見下節 |
| **個股明細抽屜**(點任一幣) | ±20 方向評分 | 做多/做空依據、評分依據明細、相關幣種 |

### BTC/ETH 選擇權深度分析(majors)— 監控儀表,非回測訊號
獨立分頁,**不影響 OI 那套**。資料來自 Deribit 免費即時選擇權鏈(BTC+ETH):
- **DVOL**(前瞻隱含波動)、**ATM IV**(近月)
- **25Δ 偏斜 RR**(call IV − put IV):>0 偏多、<0 偏空/避險需求
- **Put/Call 比**(OI):>1 避險偏空
- **Max Pain**(到期日價格「磁吸位」)、近月到期日
- **IV 期限結構**(各到期 ATM IV)、**Call/Put OI 牆**(壓力/支撐位)

> ⚠️ 明確標注:**選擇權無免費歷史 → 無法回測**,此頁是給人**主觀判讀主力幣方向/波動**的儀表,**不是回測過的自動訊號**。

## 七、過濾層 / Overlay(套在「方向訊號」呈現上)

兩個**經回測驗證**的過濾,**純前端**、可獨立開關、複合套用。**只影響推薦卡 + 數據訊號的呈現,不改評分器、不碰雷達。**

> **開關位置**:頁面**頂部列右側**兩顆按鈕「順勢過濾 ✓/✕」「OI收縮過濾 ✓/✕」。
> **狀態保存**:各自存於瀏覽器 `localStorage`(`regimeFilter` / `qualityFilter`),重新整理/重開仍記得,兩者**預設開**。
> **生效範圍標示**:數據訊號分頁標題會顯示目前生效的過濾(「順 BTC 趨勢」「OI 收縮」);關閉時推薦卡會顯示被過濾的提示(如「BTC 偏空 · 已過濾做多訊號」)。旁邊另有「BTC 趨勢:偏多/偏空」指示燈。

### 1. 順勢過濾(BTC regime)— 預設開
- **規則**:只接「順 BTC 24h 趨勢」的方向訊號(BTC 綠→只做多、BTC 紅→只做空)。
- **依據**:回測逆 BTC 趨勢訊號**淨期望為負**(−1.35% 等);順勢明顯較好。樣本內驗證。

### 2. OI 收縮過濾(品質 gate)— 預設開
- **規則**:只接「**OI 在收縮**(OI 1h < 0)」時的訊號。
- **依據**:**樣本外驗證通過**——OI 收縮(衰竭/平倉)時訊號最賺;OI 擴張(新倉湧入)時幾乎打平(逆勢追高接刀)。
- **品質徽章**:
  - **★優質** = OI 收縮 + 費率極端(樣本外最佳組)
  - **OI↓** = OI 收縮(較可靠)
  - **OI↑** = OI 擴張(追高風險)

> 本質:這兩層是**「在爛行情保護你」**——把虧損壓成打平,不是讓訊號變神。

## 八、模擬倉的搭配差異(紀律 vs 賭博)

兩本帳同時跑、即時 A/B(`internal/cache/paper.go`):

| | **紀律倉**(paper) | **賭博倉**(gamble) |
|---|---|---|
| 點火門檻 | **55** | **45**(低門檻) |
| 進場條件 | **需新鮮穿越**(剛啟動已高的不追) | **不需穿越**(直接追已高的) |
| 冷卻 | **4 小時**(方向獨立) | **1 小時** |
| 進場價 | 訊號當下**市價** | 同左 |
| TP / SL | 雷達 fib(0.618R / 0.5R) | 同左 |
| 出場 | TP / SL / 反向訊號平倉 / 24h 逾時 | 同左 |
| 紀錄 | 進場/出場時間、持倉時長、損益、結果 | 同左 |

設計目的:用真實前向資料驗證「紀律 vs FOMO」誰賺。

## 九、回測知識庫(累積至今)

回測工具 `cmd/backtest`:**重放線上同一套評分器**(零漂移),滾動 ~20-25 天。
**核心紀律:回測先行、樣本外驗證、有效才納入、沒用也記原因。**

### ✅ 已採用
- 統一評分器(修掉「負分卻標強力做多」的雙評分器 bug)
- OIHalf 1.5(原 0.3 對 1h OI 雜訊過度反應)
- 流動性抑制、真實 CVD、薄量門檻 100M
- 動能24H、費率反指標、多空比反指標
- 雷達用 **OI 方向**判方向、TP0.618/SL0.5
- **BTC 順勢過濾**(overlay)
- **OI 收縮品質 gate**(overlay,樣本外通過)

### ❌ 停用 / 否決(附原因)
| 項目 | 原因 |
|------|------|
| 1H 動能、相對強弱 | **反指標** |
| ATR、區間壓縮(squeeze) | 當因子無效 |
| 費率 z 分數、大戶比、溢價、現貨 CVD | 無效 |
| OI 加速度、CVD 斜率、多時間框架確認 | **全部反向**(印證評分器是反轉型) |
| 市場廣度過濾 | 不穩、與 BTC regime 重疊 |
| 高槓桿密度 + 紐約盤(合併) | **樣本外過擬合**(更差) |
| 時段獵殺(ICT 銀彈) | 不盡理想,已移除 |
| 快速剝頭皮、社群/DEX 資料 | 扣費後虧損 / 覆蓋率假高 |
| 跨所 OI 一致性 | OKX 無 OI 歷史,**免費 API 測不了** |

### 🔑 最大發現
**這套評分器本質是「反轉/軋空引擎」**:在「**OI 收縮 + 費率極端 + CVD 背離 + 波動偏高**」時最準;在「OI 擴張 + 費率溫和 + 市場平靜(低波動/低 IV)」時最廢。所有「動能確認類」因子都反向,完全自洽。

### ⏳ 已驗證、未上線(待定)
- **DVOL / ATR 低波動 = 廢**:可當全市場 regime 過濾,但與時間段混淆,需更謹慎。
- **CVD 背離、區間壓縮**:方向對但弱/不穩,當參考。

## 十一、待辦 / 未來方向

1. **Snapshot 落地**(SQLite 或 JSONL):每 15 分鐘存原始評分輸入 + 價格,往未來累積→突破 30 天牆→**跨行情驗證**(目前所有結論的根本限制)。
2. **上伺服器**:Linode Nanode 1GB($5)足夠;**本地 build、只上傳產物**;真正風險是 **Binance 封機房 IP**(需放慢雷達、合併 all-tickers、降併發、429 退避)。
3. 監控顯示候選:OI×價格四象限、波動儀表(ATR%/DVOL regime)、費率極端標記。
4. **代幣解鎖 tab**:卡在資料——DefiLlama 已付費化、免費歷史/即時都 gate。需申請免費金鑰(CoinMarketCal)或付費 API 才能做。
5. **清算 tab**:Binance 歷史端點停用、無免費歷史 → 只能 websocket 往後累積,暫緩。

### 資料可行性備忘(實測 2026-06)
| 想做的 | 免費歷史(可回測) | 免費即時 | 結論 |
|------|------|------|------|
| 代幣解鎖 | ❌ 付費 | ❌ 付費/爬蟲 | 需金鑰 |
